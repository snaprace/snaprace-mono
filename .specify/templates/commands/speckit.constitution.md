# /speckit.constitution 명령어

SnapRace 프로젝트 헌법을 생성하거나 업데이트하는 명령어입니다.

## 사용법
```bash
/speckit.constitution
```

## 설명
이 명령어는 SnapRace 프로젝트의 헌법을 생성하거나 업데이트합니다. 헌법은 일론 머스크의 5단계 엔지니어링 원칙을 기반으로 프로젝트의 개발 철학과 기술 표준을 정의합니다.

## 헌법 원칙

### 1. 요구사항 검증 (Question Requirements)
- 모든 요구사항이 근본적인 진실에 대해 검증되어야 함
- 똑똑한 사람들의 초기 가정은 더욱 집중적인 검토가 필요함
- "왜 이것이 존재하는가"에 대한 명확한 정당성 필요

### 2. 불필요한 것 제거 (Eliminate Unnecessary)
- 제거가 추가보다 더 중요함
- 복잡성은 신뢰성과 유지보수성의 적임
- "만약을 위해" 코드나 기능 없음

### 3. 단순화 및 최적화 (Simplify & Optimize)
- 최적화보다 단순화에 집중
- 단순한 시스템이 이해, 디버깅, 발전이 더 쉬움
- 주니어 개발자가 읽을 수 있는 코드 작성

### 4. 생산 가속화 (Accelerate Production)
- 처음 세 원칙이 적절히 검증된 후에만 속도 향상
- 정확성 없는 속도는 낭비를 만듦
- 올바른 것을 빠르게 구축하는지 확인

### 5. 자동화 (Automate)
- 마지막 단계에서만 자동화 도입
- 조기 자동화는 생산 지옥으로 이어짐
- 자동화는 효율성을 증폭시켜야 함

## 템플릿 동기화
헌법이 업데이트되면 다음 템플릿들도 자동으로 동기화됩니다:
- spec-template.md: 기능 명세서 템플릿
- plan-template.md: 구현 계획 템플릿
- tasks-template.md: 작업 관리 템플릿

## 기술 표준
- TypeScript (strict mode)
- Next.js 프레임워크
- Turborepo 빌드 오케스트레이션
- pnpm 패키지 관리
- Jest + React Testing Library (80%+ 커버리지)
- ESLint + Prettier

## 거버넌스
- **수정 절차:** 제안 → 3일 검토 → 2/3 투표 → 30일 유예
- **버전 관리:** 메이저/마이너/패치 시맨틱 버전닝
- **준수 검토:** 월간 자동화, 분기별 수동, 연간 전체 평가

## 성공 지표
- 기술: 커버리지 >80%, 빌드 <2분, 번들 <250KB
- 사용자: 검색 정확도 >95%, Core Web Vitals "Good", 만족도 >4.5/5
